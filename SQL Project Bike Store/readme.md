This project focuses on extracting business insights from a relational database for a retail bike store. Using a series of SQL queries, I analyzed sales performance, staff productivity, and inventory levels to help drive data-driven decision-making.

The project utilizes a relational schema consisting of 9 tables organized into two main scopes:

1. Sales: Customers, Orders, Order Items, Staffs, and Stores.

2. Production: Categories, Products, Stocks, and Brands.

In this project, you will find SQL solutions for:

1. Sales Performance: Calculating total revenue by category and store for specific fiscal years.

2. Staff Productivity: Identifying top-performing employees based on order volume using Window Functions (RANK()).

3. Customer Insights: Filtering high-value customers who have placed more than 5 orders using HAVING clauses.

4. Inventory Management: Tracking stock levels across different store locations.
